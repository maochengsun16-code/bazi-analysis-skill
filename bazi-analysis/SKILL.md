# 八字分析技能

使用本技能排盘、分析、解释八字命盘。把本体系视为结构化解释框架，不要当作科学、医疗、法律或财务事实。

保持 `SKILL.md` 稳定。把它作为入口和流程分发文件使用。详细知识、框架和速查资料放在 `references/` 中，按用户问题所需加载。

## 核心流程

1. 收集或校验输入。
   - 完整排盘需要：历法、年、月、日、时/分、性别。
   - 如果缺少出生时辰，说明时柱和起运时间的置信度较低。
   - 如果出生地明显偏离北京时间，或出生时间接近两个时辰的交界，提醒真太阳时可能改变时柱。
   - 如果用户已经提供四柱，可以直接使用，但若需要看大运，仍要询问性别。

2. 生成或接收命盘与固定分析输入。
   - 优先使用 `scripts/chart_cli.py` 做确定性排盘。
   - 除非用户另有指定，否则使用 `sect=2` 和 `yun_sect=1`。
   - 如果命盘由用户提供，分析前先校验日主和月支。
   - 先让程序生成固定基础层：四柱、大运、流年、十神矩阵、月令十神、根气摘要、寒燥标记、七杀标记、四墓库标记。
   - 在 JSON 输出中，使用 `analysis_prep` 作为确定性排盘到解释性分析之间的结构化桥梁。

3. 按标准顺序分析。
   - 程序预处理：排四柱、大运流年、十神体系、月令十神、根气、寒燥、七杀、四墓库。
   - 定日主，明五行。
   - 定月令十神，判断环境定调。
   - 扫描高优先级特殊分支：库做功、强七杀、寒燥调候。
   - 定格局：财、官杀、印、食伤、比劫禄刃、墓库、杀印相生、食伤制杀等。
   - 判旺衰，看根气。
   - 取用神，定喜忌。
   - 看天干地支作用关系与原局偏枯。
   - 分项断事。
   - 依次分析大运，再分析流年。

4. 应用实战派规则。
   - 日主担事能力看根气，不按传统“得令就等于身强”直接判断。
   - 根气只看地支三类：明现比劫、明现的印，现的印对应库。中气根和余气根不算有根气，只能作为藏干线索。
   - 戊己土特殊：辰戌丑未为土之比劫根；巳午火是戊己土的印，由于巳午火是戊己土的阳刃/禄，也可以看成戊己土的比劫，为强根。
   - 月令本气十神是最强十神权重，也是原局分析的主轴之一。
   - 用神通过制化最强的问题力量来取，但特殊分支可以高于普通月令取用。
   - 原局结构优先级：四墓库做功 / 强七杀 / 寒燥调候 > 月令十神格局 > 日主根气 > 普通十神组合 > 神煞。
   - 四墓库做功：如果形成墓库做功机制，先于普通月令格局分析。
   - 有杀先论杀：强七杀先于普通格局结论处理。
   - 寒燥调候：亥子丑冬令寒重先看火；巳午未夏令燥重先看水。再检查调候五行是否出现、有根、受保护或受损。
   - 地支关系看力量，月支优先，月令力量最大。
   - 穿/六害按直接伤害处理，通常比单纯冲更紧急。
   - 四墓库做功是重要财富机制。
   - 大运以地支为重，不拆成前五年和后五年。
   - 专题隔离原则：分析某一专题时，只使用与该专题直接相关的十神、宫位、做功链和运年作用。财运分析只看事业、财富、工作、经营、投资、现金流、债务、回款等事项；不得引入婚姻、健康、父母、子女、学业结论来否定财运吉凶。若同一大运或流年在其他专题有不利作用，应另列专题说明，表达为“财运维度为吉，但婚姻维度有动荡”，不能合并成“此运不好”。
   - 神煞只作辅助，绝不能覆盖五行、十神、根气、地支关系或用神。

5. 标注置信度。
   - 使用“高”：有明确核心规则或多个独立证据支持，且没有未消解冲突；如果有反向验证通道，反向验证也应通过。3 条及以上规则一致，可以增强到“高”，但不是唯一入口。
   - 使用“中”：有规则支持，但需要权衡；证据质量尚可但样本不足；存在轻微冲突但可以通过优先级、力量或适用条件解释。标注“中”时说明原因：是证据不足、样本不足、需要权衡，还是存在已消解冲突。
   - 使用“低”：缺少出生时辰、真太阳时不确定、证据薄、规则链过长，或出现无法充分消解的冲突。
   - 使用“极低”：关键输入缺失、具体年份强断、具体财富数字、医疗/法律/财务确定性结论，或结论主要依赖高度不确定的推断。
   - 即使是高置信度，也避免绝对化预测。健康、法律、财务、婚姻类结论要用“明确倾向”“大概率”“结构上很明显”等表达，不要用“一定”“肯定”当作事实断言。


## 输出格式

普通用户命盘解读按以下结构输出：

1. 基础命盘摘要：四柱、日主、月支、月令十神、当前大运。
2. 程序准备的基础层：十神矩阵、根气摘要、寒燥倾向、七杀标记、墓库标记。
3. 特殊优先级扫描：活跃墓库做功、强七杀、寒燥调候。
4. 核心结构：月令环境、根气强弱、主要格局、用神忌神。
5. 关键天干地支关系与原局偏枯。
6. 先回答用户指定主题，再补充高置信度提醒。
7. 大运流年应期，必须在原局固定后再分析；若用户只问单一专题，只输出该专题的大运流年结论，不混入其他专题。
8. 说明限制：缺少时辰、真太阳时风险或命理判断本身的非确定性。

智能体内部工作时，保持如下结构化草稿对象：

```json
{
  "day_master": "戊土",
  "month_ling": "子",
  "month_ling_shishen": "正财",
  "priority_scan": ["寒重调候", "强七杀"],
  "patterns": ["伤官配印"],
  "useful_gods": [{"element": "火", "shishen": "印星"}],
  "harmful_gods": [{"element": "土", "shishen": "比劫"}],
  "day_master_root": "有根",
  "season_bias": {"type": "寒重", "useful_element": "火"},
  "tomb_work": ["丑未冲开库"],
  "qisha": {"exists": true, "strong": true},
  "special_combinations": ["杀印相生"]
}
```

将 `day_master_root` 作为稳定结构化字段，只使用 `有根` 或 `无根`。较长的根气解释文本保留在 `analysis_prep.root_summary.conclusion` 中。
